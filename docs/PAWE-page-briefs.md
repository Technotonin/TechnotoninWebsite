# PAWE — Page Briefs

Content briefs for each page. Fill in each section before building.

---

## Home (`index.html`)

**Purpose:** First impression. Capture waitlist signups. Establish what PAWE is.

**Sections:**
1. **Hero** — Headline + subtext + primary CTA ("Join the waitlist")
2. **Value props** — 3 stat callouts (e.g., 60 seconds, weight, range)
3. **How it works** — Brief overview + link to full page
4. **Social proof** — Trust signals (beta users, partners, press — when available)
5. **Final CTA** — Repeat waitlist signup

**Headline direction:** Lead with the user outcome, not the feature.

---

## How It Works (`how-it-works.html`)

**Purpose:** Explain the product clearly. Build confidence that it's simple and real.

**Sections:**
1. **Page header** — "How it works" + brief intro
2. **Step-by-step** — 3 steps with visuals (attach, go, detach)
3. **Technical specs** — Weight, range, speed, battery, compatibility
4. **CTA** — "Join the waitlist" or "See it in action"

---

## For Providers (`for-providers.html`)

**Purpose:** B2B credibility. Drive Letters of Intent from DME suppliers, rehab hospitals, VA.

**Tone:** More formal than other pages. Lead with clinical/operational value.

**Sections:**
1. **Page header** — Speak directly to the provider audience
2. **Value to providers** — Reimbursement potential, patient outcomes, ease of deployment
3. **Specs summary** — Key technical details relevant to procurement
4. **LOI CTA** — "Request more information" or "Express interest"

---

## About (`about.html`)

**Purpose:** Tell the founding story. Build trust with both users and investors.

**Sections:**
1. **Mission statement** — Why PAWE exists
2. **Founding story** — Brief narrative
3. **Team** — Founders + key team (photos when available)
4. **CTA** — "Join the waitlist" or "Get in touch"

---

## Waitlist (`waitlist.html`)

**Purpose:** Pure conversion. Minimal distractions. Collect signups.

**Sections:**
1. **Headline** — Reinforce the value prop
2. **Form** — Name, email, "I am a..." (user / caregiver / provider / other)
3. **Reassurance** — "No spam. We'll only email when it matters."

**Design:** Centered layout, no sidebar, no competing CTAs.
